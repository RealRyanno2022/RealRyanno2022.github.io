Spring Boot and Kotlin

Full-stack server architecture

  • Repository:

    • Context

    I think that every full-stack developer should be able to make a mobile application of sorts. My first mobile framework was React Native. I would like to acknowledge Diarmuid McGonagle, a fellow developer, who encouraged me to learn the language. Kotlin is like, in my opinion, "Better Java". I also saw memes about how start-up engineers could set up a Spring Boot server, a database, frontend, Kubernetes cluster and node backend in the same time it takes a FAANG engineer to take a coffee break. So I decided to make a Kotlin project which is hosted on a Spring Boot server. The application is a blog which allows you to post your own articles. This application would be capable of running on Android devices, and because Kotlin is quite easy to convert to Swift code, it could also be ported to iOS devices. I could achieve this by completing a few Swift tutorials and buying a Mac to run XCode. Overall I found SpringBoot to be a very accessible server framework and would be open to using it in the future. However, Kotlin is awkward because it has to be written in Android Studio and relies on a working Gradle configuration. I actually spent more time working around Gradle than writing Kotlin. That is why I will stick with React Native.
