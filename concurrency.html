Python Concurrency

Multithreaded Applications

Context

Concurrency is the process of spreading the load of a computation across several application "threads" which are running simultaneously. This project involved taking a short course in concurrency which focused on Python architecture. I learnt about CPU-bound and IO-bound concurrency, as well as parallelism - where multiple processes each run in their own Python interpreter. An IO bound problem exists because of delays in input or output while a CPU bound problem occurs when you are working on problems that are slower than your CPU. I compared conventional synchronous IO bound code with asynchronous threaded code. I also learnt about the GIL (Global Interpreter Lock) which is a mutex that prevents multiple threads from executing Python bytecodes at once. The GIL helps prevent crashes and obtuse behaviour. I also compared conventional synchronous CPU-bound code with asyncio or multiprocessing versions. The program is tasked with simulating a large number of dice rolls. The multiprocessing version downloads all the possible sites from the Jython website, a Javascript-Python website which can be called directly by Python code. The asyncio version rolls the die asynchronously. This project proved immensely beneficial in helping me learn the inner workings of Python.
